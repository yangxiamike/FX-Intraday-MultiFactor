# 外汇单标的多因子研究流程

版本：`v0.2`

状态：`Draft`

注：

- 第 1 轮研究与测试默认基于 Massive/Polygon 免费档 API 获取并固化的真实样本
- 不再使用生成假行情作为正式研究验证输入

## 1. 适用范围

本文适用于外汇单标的时序因子研究，核心问题是：

- 这个标的当前是否值得交易
- 因子能否稳定预测未来收益或方向
- 因子在成本后是否仍有交易价值

单标的研究默认看时间序列预测能力，因此以 `IC`、命中率、净收益和稳定性为主，不强调 `Rank IC`。

## 2. 研究输入

每次研究先固定以下内容：

- 标的：如 `USDJPY`
- 频率：如 `1m`、`5m`、`1h`
- 持有期：如未来 `5 bars`、`15 bars`、`1d`
- 标签：未来收益、方向或波动
- 成本口径：`spread + slippage + carry`

建议优先使用净收益标签：

```text
net_return_t = close[t+H] / close[t] - 1 - spread - slippage - carry
```

## 3. 数据要求

至少准备：

- OHLCV 或报价数据
- 点差或点差代理
- carry
- 统一时区时间戳
- session 标签
- 重大事件标签

最低检查项：

- 时间戳连续
- 无重复 bar
- 无明显脏值
- 因子只使用时点可得信息
- 标签严格向后看

## 4. 因子构造

先按简单类别推进：

- 趋势类
- 反转类
- 波动类
- 微观结构类
- 状态类

每个因子建议保留：

- 原始值
- 标准化值
- 波动调整值

常用处理：

```text
z_t = (x_t - mean_{t-N:t-1}) / std_{t-N:t-1}
scaled_t = signal_t / rolling_vol_t
```

核心原则是和“自身历史”比，不是和其他资产比。

## 5. 标准化、中性化与分块分析

这三者不要混用。

### 5.1 标准化

单标的外汇因子默认应做时间序列标准化，用于提升不同时间段的可比性，常见方式包括：

- rolling z-score
- rolling vol scaling
- 极端值截尾

标准化是默认动作。

### 5.2 中性化

单标的外汇时序研究通常不需要中性化。

原因是单标的任务不是横截面排序，不存在股票研究里典型的：

- 行业暴露
- 市值暴露
- 风格暴露

因此，`session`、`regime`、交易日历、事件标签一般不作为中性化对象。

只有在以下场景，中性化或暴露剥离才有意义：

- 多货币对横向分配
- 组合层想剥离美元整体暴露
- 组合层想控制某类公共风险暴露

如果当前研究对象是单货币对时序因子，默认结论是：

- 做标准化
- 不做中性化

### 5.3 分块分析

`session`、`regime`、交易日历更适合做分块分析和条件启用，不适合默认做中性化。

它们的主要作用是：

- 解释因子在哪些环境下有效
- 识别失效区间
- 形成条件启用规则
- 避免把不同分布的数据混在一起评估

因此，单标的外汇研究里更推荐：

- 分块分析
- 状态过滤
- 条件启用

而不是：

- 对这些变量做中性化回归后再研究因子

## 6. 样本切分

不能随机切分，必须按时间切分：

- 训练集
- 验证集
- 测试集
- walk-forward

同时建议按以下状态拆分：

- 高波动 / 低波动
- 趋势 / 震荡
- 亚洲盘 / 欧盘 / 美盘
- 事件窗口 / 非事件窗口

## 7. Session 与 Regime 的定义

先验证预测能力，再验证交易价值。

### 7.1 Session

`session` 是交易时段标签，按时钟切分。它的目标不是解释宏观，而是区分不同流动性、波动和交易活跃度环境。

建议先固定一版简单规则，例如以 `UTC` 切分：

- `asia`
- `london`
- `overlap`
- `new_york`
- `rollover`

关键要求：

- 一旦定义，尽量长期保持一致
- 先服务于研究可比性，不追求完美市场叙事

### 7.2 Regime

`regime` 是市场状态标签，按价格行为和波动特征切分，不按主观叙事切分。

建议先做两类基础标签：

- 波动状态：`low_vol / high_vol`
- 趋势状态：`trending / ranging`

常见实现方式：

- 用 rolling volatility 或 ATR 分位数定义波动状态
- 用 ADX、均线斜率、累计收益相对波动比定义趋势状态

第一版不建议把 regime 做复杂，能稳定复现即可。

### 7.3 交易日历与事件标签

交易日历和事件标签用于识别特殊窗口，例如：

- 非农
- CPI
- 央行议息
- 节假日薄成交时段
- rollover 附近时段

这些标签更适合：

- 过滤
- 单独统计
- 条件启用

而不是和普通样本混在一起做统一结论。

## 8. 单因子评估

核心指标只保留：

- IC
- 命中率
- 因子衰减
- 样本外表现
- 成本前后净收益
- 年度稳定性
- regime 稳定性

拒绝标准：

- 收益只集中在少数阶段
- 参数轻微变化就失效
- 成本后失效
- 只在单一 regime 有效但没有明确条件启用逻辑

## 9. 分块分析要求

单因子评估和组合评估都应输出分块结果。

建议至少按以下维度分块：

- `session`
- `vol_regime`
- `trend_regime`
- `event_flag`

建议至少查看以下结果：

- 各分块下的 IC
- 各分块下的命中率
- 各分块下的平均净收益
- 各分块下的样本数

有条件时可增加交叉分块：

- `session x trend_regime`
- `session x vol_regime`

分块分析的目标不是追求切得越细越好，而是回答两个问题：

- 因子在哪些环境下有效
- 因子在哪些环境下应关闭

如果因子只在特定分块下有效，研究结论应明确写成条件因子，而不是普适因子。

## 10. 多因子合成

单因子通过后再做组合。

原则：

- 先看样本外，再看样本内
- 先保留经济含义清晰的因子
- 控制高相关冗余
- 从简单加权开始

推荐顺序：

1. 等权
2. 按 IC 或稳定性加权
3. 条件启用
4. 再考虑模型化方法

组合层重点看：

- 净值曲线
- 年化收益
- Sharpe
- 最大回撤
- 换手
- 成本敏感性
- 与已有组合的相关性

## 11. 回测要求

区分两层：

- 研究型回测：验证方向、阈值、持有期、成本后期望
- 订单级回测：纳入真实成本和交易约束

最低输出：

- 成本前后净值对比
- 分年份表现
- 分 session 表现
- 分 regime 表现
- 回撤分析

## 12. 稳健性检查

每个因子至少做：

- 参数扰动
- 持有期扰动
- 频率扰动
- 成本上调测试
- 训练窗口变化
- 去除极端事件窗口再测试

如果轻微改动就明显失效，默认视为不稳健。

## 13. 常见坑

### 10.1 把趋势行情当成稳定因子

解决方法：

- 分年份看
- 分趋势 / 震荡看
- 做 walk-forward
- 看收益是否过度集中

### 10.2 忽略交易成本

解决方法：

- 从研究开始就纳入成本
- 用保守点差假设
- 先看单笔优势能否覆盖成本

### 10.3 不做 regime 拆分

解决方法：

- 给样本打 regime 标签
- 分状态看 IC、命中率和净收益
- 必要时改成条件因子

### 10.4 用过长历史训练当前市场

解决方法：

- 优先 rolling window
- 比较不同训练窗口
- 更重视近样本

### 10.5 因子冗余

解决方法：

- 看相关性
- 做逐步加入测试
- 只保留有边际贡献的因子

## 14. 入池标准

因子进入候选池前，至少满足：

- 样本外 `IC > 0` 或命中率稳定高于随机
- 成本后净收益为正
- 多个年份不过度依赖单一年份
- 参数不过度敏感
- 对失效区间有清晰解释

## 15. 最低纪律

- 不随机切分时间序列
- 不用测试集选参数
- 不忽略成本讨论 alpha
- 不只看全样本结果
- 不只凭一次回测决定入池

满足这些约束，研究结果才有继续进入组合和实盘验证的价值。
